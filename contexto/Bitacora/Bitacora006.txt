--- 7.3 — FIX PdfController + AGREGAR ACCIÓN PDF EN LAS 3 TABLAS -----

  BUG CRÍTICO: Cliente NO podía descargar PDF de SUS documentos
  (can('venta.ver') == false para rol cliente).

  Solución — agregar helper privado con doble check:
    private function puedeVerDocumento(mixed $documento, string $permisoStaff, string $permisoPortal): bool
    {
        $user = Auth::user();
        if (! $user) return false;
        if ($user->can($permisoStaff)) return true;          // admin path (compat)
        if ($user->can($permisoPortal)) {                    // portal path
            $clienteId = $user->cliente?->id;
            return $clienteId !== null
                && (int) $documento->cliente_id === (int) $clienteId;
        }
        return false;
    }

  Aplicado en PdfController::venta, ::remision, ::cotizacion.
  Backward compatible: comportamiento admin NO cambia.

  Acción de impresión en las 3 tablas (reutiliza PrintAction existente):
    PrintAction::make('pdf.venta')       // en FacturaPortalsTable
    PrintAction::make('pdf.remision')    // en RemisionPortalsTable
    PrintAction::make('pdf.cotizacion')  // en CotizacionPortalsTable
