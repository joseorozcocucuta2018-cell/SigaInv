SESIÓN 04 — 2026-03-04 — INVESTIGACIÓN DEFINITIVA AVATARES + CHECKLIST SEGURIDAD
================================================================================

INVESTIGACIÓN DEFINITIVA: Avatar de usuario vs Logo de empresa
  SÍNTOMA: Logo de empresa funciona, avatar de perfil de usuario no aparece.

  ANÁLISIS PROFUNDO (Sesión 4):
    Se inspeccionó el filesystem real del proyecto:
    
    storage/app/private/avatars/   ← archivos reales de avatares
    storage/app/private/empresa/   ← archivos reales de logos
    storage/app/public/            ← VACÍO (solo .gitignore)
    public/storage/                ← era directorio vacío, NO symlink
    
    CAUSA RAÍZ REAL:
    1. FILESYSTEM_DISK=local en .env → todos los uploads van a disco 'local'
       → disco 'local' = storage/app/private/
    2. filament-edit-profile usa env('FILESYSTEM_DISK', 'public') = 'local'
       → guarda avatares en storage/app/private/avatars/ ✅
    3. EmpresaForm no especifica disco → usa Filament default = env('FILESYSTEM_DISK','local') = 'local'
       → guarda logos en storage/app/private/empresa/ ✅
    4. UserResource tenía ->disk('public') EXPLÍCITO en FileUpload, ImageColumn, ImageEntry
       → buscaba archivos en storage/app/public/avatars/ → NO EXISTEN → imagen rota ❌
    5. getFilamentAvatarUrl() tenía Storage::disk('public')->url() EXPLÍCITO
       → generaba URL para disco 'public' → archivos no están ahí → 404 ❌
    
    POR QUÉ EMPRESA FUNCIONA Y USUARIO NO:
    - Empresa: disco 'local' (default) → serve: true → Laravel sirve internamente ✅
    - Usuario: disco 'public' (explícito, incorrecto) → necesita symlink + archivos en public ❌
    
    CORRECCIONES APLICADAS (Sesión 4):
    
    1. app/Models/User.php:
       ANTES: Storage::disk('public')->url($this->avatar)
       AHORA:  Storage::url($this->avatar)  ← usa disco default (local)
    
    2. app/Filament/Resources/UserResource.php:
       ANTES: FileUpload → ->disk('public') ->visibility('public')
       AHORA:  FileUpload → sin disco explícito (usa default = local)
       
       ANTES: ImageColumn → ->disk('public')
       AHORA:  ImageColumn → sin disco explícito
       
       ANTES: ImageEntry → ->disk('public')
       AHORA:  ImageEntry → sin disco explícito
    
    3. Symlink storage:link:
       - public/storage era directorio vacío (no symlink)
       - Se eliminó el directorio y se ejecutó: php artisan storage:link
       - Resultado: public/storage → storage/app/public (junction creada)
       - Nota: el symlink es para el disco 'public', que actualmente no se usa
         para avatares, pero es buena práctica tenerlo para el futuro
    
    RESULTADO ESPERADO:
    - Avatares: disco 'local' → serve: true → URL servida por Laravel ✅
    - Logos empresa: disco 'local' → serve: true → URL servida por Laravel ✅
    - Consistencia total: todos los uploads usan el mismo disco ✅

4. LÓGICA DE SALDO CLIENTE
   
   IMPACTO EN cliente.saldo:
     - Antes: saldo = deuda del cliente (positivo = debe dinero)
     - Al devolver: saldo se reduce por el monto devuelto
     - Si saldo < 0: cliente tiene crédito a favor (saldo negativo)
   
   EJEMPLO:
     Cliente compra 100,000 → saldo = 100,000 (debe)
     Devuelve 30,000 → saldo = 70,000 (debe reducido)
     Devuelve 80,000 → saldo = -10,000 (tiene 10mil a favor)
     Compra nueva por 15,000 → saldo = 5,000 (con el crédito pagó parte)
   
   AUDITORÍA:
     Cada cambio registrado en movimientos_saldo_cliente:
       - Referencia: "devolucion_123"
       - saldo_anterior: 100,000
       - saldo_nuevo: 70,000
       - monto: -30,000 (negativo = reducción de deuda)
       - descripción: "Devolución DVL-00001 - defecto"
