--- DECISIONES CLAVE DE LA SESIÓN -------------------------------------------

  1. Cliente implementa FilamentUser (con canAccessPanel) en vez de
     middleware EnsureClientePortalActivo. Filament v5 Authenticate
     middleware aborte 403 en entorno no-local si el modelo no
     implementa FilamentUser (verificado en
     vendor/filament/filament/src/Http/Middleware/Authenticate.php).

  2. Rutas PDF/Excel: ['auth:web,cliente'] para que el guard 'cliente'
     acepte al Cliente. ['auth'] solo acepta el guard web.

  3. PdfController::puedeVerDocumento detecta $user instanceof User
     para staff (usa can()) y Auth::guard('cliente')->user() instanceof
     Cliente para clientes (comparación directa por cliente_id, sin
     permisos).

  4. Helper de test crearClienteConAcceso pasa password en crudo (no
     Hash::make) porque el cast 'hashed' del modelo lo hashea
     automáticamente. Pasar Hash::make producía doble-hash y los
     tests de login fallaban.

  5. Mailable con ShouldQueue (no bloquea), password plano como
     propiedad pública para la vista. La vista no loguea el password.

  6. Seeder ClientesPortalSeeder NO se agrega a DatabaseSeeder — es
     one-shot que el usuario corre manual con
     `php artisan db:seed --class=ClientesPortalSeeder`.

  7. user_id_portal queda en la tabla (nullable) y en la migración
     principal (no se borra la columna) — solo se quita de fillable
     y se elimina la relación BelongsTo.

  8. Cliente::getFilamentName() requerido para evitar TypeError en
     Filament\FilamentManager::getUserName() (que llama
     $user->getAttributeValue('name') por defecto — no existe en
     Cliente que tiene 'nombre').

  9. Cliente usa Notifiable trait porque el panel tiene
     ->databaseNotifications() que llama notifications() en el user.

  10. ClienteForzarCambioPassword middleware fuerza redirect a
      cambiar-password cuando password es temporal — añadido al
      ->authMiddleware() del panel. Authenticate middleware ya filtra
      no autenticados, así que solo se ejecuta para clientes logueados.

  11. Helper crearClienteConAcceso usa bool $temporal (no DateTime|null)
      para distinguir "password definitivo" de "password temporal".
      La versión anterior con $passwordChangedAt=null colapsaba a now()
      con `?? now()` y rompía el test de "password temporal".

  12. CambiarPassword NO overridea render() — usa $view property y deja
      que BasePage::render() use el layout filament-panels por defecto
      (layouts/index). Si se overridea render() sin ->layout() Livewire
      cae en `layouts::app` que no está registrado en este proyecto.

--- BUGS DESCUBIERTOS Y ARREGLADOS DURANTE 8.15 -----------------------------

  1. getUserName() TypeError — Filament esperaba atributo 'name',
     Cliente tiene 'nombre'. Fix: implements HasName + getFilamentName().

  2. layouts::app not defined — CambiarPassword render() custom sin
     layout. Fix: remover render() y usar $view property.

  3. Call to undefined method Cliente::notifications() — el panel
     databaseNotifications() lo requiere. Fix: use Notifiable trait.

  4. Test "password temporal" no redirigía — el helper usaba
     $passwordChangedAt ?? now() que colapsaba null a now(). Fix:
     cambiar firma a bool $temporal con default false.
